### **Must-Have (Kritisch für den Launch)**

**1. Automatischer Lernpfad-Generator**

-   **Story:** Als **Lena** möchte ich meine Vorlesungsfolien hochladen, damit ich sofort einen strukturierten Lernplan erhalte, ohne selbst planen zu müssen.

-   **AC 1:** Given 15 PDF-Folien When der Upload abgeschlossen ist Then wird in unter 30 Sekunden eine Liste mit Lernmodulen angezeigt.

-   **AC 2:** Given ein generierter Plan When der Prüfungstermin eingegeben wird Then werden die Module auf die verbleibenden Tage verteilt.

**2. Dynamische Budget-Anpassung**

-   **Story:** Als **Lena** möchte ich, dass sich mein tägliches Lernpensum automatisch anpasst, wenn ich einen Tag verpasse, damit ich trotz Zeitnot alle Kernkonzepte abdecke.

-   **AC 1:** Given ein bestehender Plan When ein Tag als \"nicht gelernt\" markiert wird Then wird der Umfang der restlichen Tasks neu berechnet (Verkürzung/Fokus).

-   **AC 2:** Given eine Budget-Änderung When der Plan aktualisiert wird Then bleiben die Kernkonzepte priorisiert erhalten.

**3. Sokratischer Abfrage-Modus**

-   **Story:** Als **Tobias** möchte ich interaktiv zum Stoff befragt werden, damit ich sicherstelle, dass ich die Konzepte wirklich verstanden habe.

-   **AC 1:** Given ein Lernmodul When Tobias den Tutor startet Then stellt die KI eine offene Transferfrage statt einer Ja/Nein-Frage.

-   **AC 2:** Given eine Antwort von Tobias When diese unvollständig ist Then gibt die KI einen Hinweis (Scaffolding), statt sofort die Lösung zu verraten.

### **Should-Have (Wichtiger Mehrwert)**

**4. Wissenslücken-Radar**

-   **Story:** Als **Tobias** möchte ich eine Übersicht meiner Schwachstellen sehen, damit ich meine begrenzte Zeit gezielt in schwierige Themen investiere.

-   **AC 1:** Given mehrere absolvierte Tutoren-Sessions When Tobias das Dashboard öffnet Then werden Themen mit schlechter Performance rot markiert.

-   **AC 2:** Given eine identifizierte Lücke When Tobias \"Fokus-Lernen\" wählt Then schlägt das System primär Inhalte zu diesem Thema vor.

**5. Spaced Repetition Integration**

-   **Story:** Als **Lena** möchte ich, dass kritische Inhalte in idealen Abständen wiederholt werden, damit ich das Gelernte nicht nach der Prüfung sofort wieder vergesse.

-   **AC 1:** Given ein erfolgreich gelerntes Modul When drei Tage vergangen sind Then taucht eine kurze Review-Task automatisch wieder im Plan auf.

-   **AC 2:** Given eine Review-Task When diese falsch beantwortet wird Then sinkt das Intervall für die nächste Wiederholung.

### **Could-Have (Zusatzfeatures)**

**6. User-Highlighting zur Priorisierung**

-   **Story:** Als **Lena** möchte ich wichtige Folien manuell markieren können, damit die KI diese im Lernplan stärker gewichtet.

-   **AC 1:** Given die Folienansicht When Lena eine Folie mit \"Wichtig\" markiert Then werden die daraus resultierenden Tasks im Plan als \"High Priority\" gekennzeichnet.

-   **AC 2:** Given markierte Folien When das Zeitbudget extrem knapp wird Then werden unmarkierte Folien zuerst aus dem Plan gestrichen.

**7. Deep-Dive für Experten (API/Export)**

-   **Story:** Als **Tobias** möchte ich meine Lernfortschritte oder generierten Karten exportieren können, um sie in meine bestehenden Workflows zu integrieren.

-   **AC 1:** Given eine abgeschlossene Einheit When Tobias \"Export\" wählt Then wird eine strukturierte Markdown-Datei erstellt.

-   **AC 2:** Given generierte Fragen When Tobias den Export startet Then ist das Format kompatibel mit gängigen Tools (z.B. Anki-Import).

### **Won\'t-Have (Vorerst nicht geplant)**

**8. Social Learning / Freundes-Vergleich**

-   **Story:** Als **Lena** möchte ich sehen, wie viel meine Freundinnen gelernt haben, um mich zu motivieren.

-   **Begründung:** Fokus liegt auf individuellem, aufwandsbasiertem Lernen und Datenschutz (Tobias\' Bedenken); Social Features erhöhen die Komplexität im Onboarding massiv.
